The **Tasks** tab on a release lets you create and manage a checklist of everything that needs to happen for a successful release.

## Creating tasks

1. Open a release and go to the **Tasks** tab
2. Click **Add Task** or press **T** as a keyboard shortcut
3. Fill in:
   - **Title** -- what needs to be done
   - **Assignee** -- which team member is responsible (optional)
   - **Due date** -- when it should be completed (optional)
   - **Priority** -- the importance level

## Working with tasks

- **Mark complete** -- check off tasks as they're done
- **Reorder** -- drag and drop tasks to change their priority order
- **Assign** -- tasks are color-coded by assignee for easy scanning
- **Bulk actions** -- mark all tasks complete or delete all at once

## Importing from templates

If you have [release templates](/templates/release-templates/) set up, you can import a full task checklist into any release:

1. Go to the Tasks tab
2. Click **Import from Template**
3. Select a template
4. All template tasks will be added to the release

This saves time when you follow a consistent release process.

## Task notifications

When notifications are enabled, team members receive alerts for:

- Being assigned to a task
- Tasks due within 24 hours
- Overdue tasks

Configure notification preferences in **Settings > Notifications**.

## Tips

- Use templates to standardize your release workflow across all releases
- Assign tasks to specific team members so nothing falls through the cracks
- The dashboard shows urgent tasks across all releases in one place
